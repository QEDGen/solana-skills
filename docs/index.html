What it does

Extracts what matters from your code and produces machine-checked proofs of your core business logic.

Reads your source

Understands Anchor IDLs, account constraints, and pure Rust logic. No manual annotation required.

Writes a verification spec

Generates SPEC.md with security goals, state model, operations, and formal property statements -- all derived from your code.

Produces Lean 4 proofs

Models your program in Lean 4, writes theorems and proofs, compiles with lake build, and iterates until everything checks. Every theorem compiles or it doesn't ship.

    (p_s p_s' : EngineState) (p_amount : Nat)
    (h_inv : conservation p_s)
    (h : depositTransition p_s p_amount = some p_s') :
    conservation p_s' := by
  unfold depositTransition at h
  split_ifs at h with h_le
  cases h
  unfold conservation at h_inv ⊢

  omega

Get started

Three steps to formally verified Solana programs.

1. Install the skill

npx skills add qedgen/solana-skills

Automatically sets up Rust, Lean/elan, the CLI binary, and a global Mathlib cache.

2. Set your Mistral API key

Free from console.mistral.ai.

export MISTRAL_API_KEY=your_key_here

3. Ask your agent to verify your program

Open your agent in your Solana project directory and say:

Formally verify this program.

The agent reads your code, writes SPEC.md, generates Lean 4 proofs, and iterates until they compile.

First Lean build takes 15-45 minutes to compile Mathlib. Subsequent builds reuse the cache.
